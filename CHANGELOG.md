### Changed

## [0.2.0-alpha.5] - 2026-07-23

### Added

- 翻页模式支持键盘左右方向键翻页；输入框、设置弹窗和中文组合输入期间不会误触发。
- Public Alpha README 已补齐安装、首次 AI 配置、备份、隐私、安全、支持范围、版本通道、已知限制和反馈入口。
- CI 新增独立 RustSec 审计，正式产物预检会拒绝测试书、`.env`、证书/私钥文件和常见私钥正文形态。
- P7.5 新增统一陪读事件、来源锚点、策略/会话状态引用和 SQLite schema 10；目录式备份支持陪读事件预览、校验与去重合并。
- 陪读消息支持安全 Markdown 富文本，并为新问题、新回答和阅读过程记录增加方向明确的轻量入场动效。
- P7.6 根据现有书页按需建立稳定正文块与文本质量判断，旧书无需重新导入；阅读进度开始区分到达、有效阅读和本节完成。
- 严格防剧透问答可使用当前页与确认已读内容；选区、笔记、聊天引用和陪读事件支持可选来源定位 v2，并兼容旧 v1 页码定位。
- P7.7 新增统一按需上下文组装与可核验来源摘要；导读、读中问答和读后回想按场景筛选材料，三档回答深度同时约束上下文、结构和输出 token。
- 章节导读支持按正文、计划连续性、策略、模型和 prompt 版本命中完整制品缓存；明确“重新生成”仍会获取新结果。
- 「整理这本书」新增跨章节记忆管理，可查看来源阅读项、修改内容或撤销记忆；旧记录只在来源明确时补齐关联，备份固定样本覆盖记忆和删除状态恢复。
- 读伴新增由真实任务驱动的静默状态：导读准备、回答、保存记录、完成、错误和离线使用一致反馈；正文阅读保持静止并支持系统“减少动态效果”。
- 设置页新增“AI 调用与选材”解释：可查看材料类型、排除原因、阅读规则、预算和缓存状态；诊断只保存脱敏引用与计数，不保存正文、笔记、回答或密钥。
- P7 新增七类固定回归案例、专项隐私测试、正式包预检和候选包人工验收清单，P7.1-P7.11 完成收口。

### Changed

- 章节翻页不再携带读伴形象平移；书页结束后读伴才从阅读区右侧浮现。
- 书籍信息页新增逐章阅读开关，并改为固定视口、章节列表内部滚动的响应式布局；书名页、目录、出版说明、自序、参考书目和附录的默认分类更准确。
- 阅读计划允许章节轻微超出单次页数上限；确需拆分时改为均匀分段，旧版产生的纯注释小尾段会自动合并并保留阅读位置。
- 章节导读改为更具体、直白的表达，并统一应用可维护的 AI 用词替代偏好。

### Fixed

- 修复书架操作菜单被书卡边界裁切、导读生成入口残留竖线且未居中，以及读伴设置步骤文案不自然的问题。
- 模型 profile 现在只能调低读伴回答上限，不能突破“简短/适中/深入”对应的硬 token 预算。
- 撤下会打断阅读的主动提问入口、原型卡和主动程度设置；读伴改为只响应用户主动发起的导读、提问、笔记与读后回想，旧事件和备份继续兼容。
- 修复模型回答中的 Markdown 粗体与列表标记被直接显示的问题；多段富文本继续支持时间线折叠与展开。
- 修复「不透露未读内容」仍可能出现后续情节预告的问题；严格防剧透会同时裁剪整本书/历史模型上下文、强化系统规则、关闭未审查流式展示，并在保存前过滤带有明确后文泄露提示的段落。
- 修复陪读时间线长问答自动折叠后无法展开的问题；最新一轮问题与回答默认展开，历史长内容可点击展开/收起，导读、问题、回答、笔记和回想使用不同视觉语义。
- 陪读时间线改为三轨对话布局：用户提问靠右、读伴回答靠左，导读、笔记、回想和本节记录居中，减少过程记录与对话气泡之间的混淆。
- 修复 MOBI/纯文本翻页模式的透明页边按钮覆盖正文、导致靠近书页边缘的短词被划成后续整段的问题；页边轻点翻页改由阅读区判断，文字拖选优先且仍保留左右边缘翻屏。
- 修复 MOBI/纯文本阅读中从划选原文创建笔记后没有显示高亮的问题；待保存笔记会即时高亮，历史笔记在滚动与自适应翻页模式中均可恢复，并继续支持取消高亮和重新划原文。
- 修复扩展名为 `.mobi`、内部同时包含旧 MOBI 与 KF8/AZW3 内容的混合电子书可能只导入 1 个残缺文本页的问题；导入器会比较可用解析结果并选择结构、目录和正文更完整的候选，同时忽略版权/目录和含子章节的卷标题，并将无标题正文片段并回上一真实章节。
- 修复 MOBI/纯文本在翻页模式下按固定文本页渲染、窗口较小时正文被底部裁断的问题；阅读器会按实际可用宽高把逻辑文本页切成若干屏，翻页入口先在当前文本页内移动，到边界后再切换逻辑页，进度、笔记和 AI 上下文页码保持不变。

### Security

- 将 `quinn-proto` 从 0.11.14 升级到 0.11.15，修复 RustSec 报告的远程内存耗尽漏洞；Tauri/plist 暂时锁定的两条 `quick-xml` advisory 使用带输入边界和移除条件的精确例外。

### Known limitations

- 当前正式 macOS 构建仅发布 Apple Silicon `arm64`。
- 扫描版 PDF OCR、PDF 图片/表格理解、MOBI 内嵌图片和自动双页仍未支持。
- Alpha.4 → Alpha.5 App 内自动更新将在本次发布后进行人工验收；升级前建议先导出一次本地备份。

## [0.2.0-alpha.4] - 2026-07-13

### Added

- 新增首次 AI 配置引导；未配置用户可按服务、密钥验证和完成三步连接模型，默认推荐 DeepSeek。
- PDF 翻页模式新增窗口宽高自适应、专注阅读、书页边缘点击、横向手势和可关闭的轻量翻页动画。
- 品牌 Logo 收束为横版、竖版和简版三种规格，并统一应用内中文字体入口。

### Changed

- PDF 阅读项优先显示“本节页码”，原书页码仅在来源和引用场景中提示。
- 窄窗口下读伴改为覆盖式侧栏，滚动和翻页模式都支持专注阅读。
- 测试与正式桌面环境继续使用独立数据目录、Keychain service 和发布通道。

### Fixed

- 桌面版历史 PDF 不再通过 macOS `asset://` 协议读取；改用受限 Tauri fs 插件从 App 数据目录读取二进制，修复 PDF.js 状态 `0` 导致的旧书加载失败。
- 测试版补齐 updater 插件初始化配置，避免桌面测试环境启动失败。
- 修复窄窗口下标题竖排、PDF 过度缩小和正文不可辨认的问题。

### Security

- 关闭 Tauri asset protocol，只保留 `$APPDATA/files/**` 范围内的单文件只读权限。

### Known limitations

- 当前正式 macOS 构建仅发布 Apple Silicon `arm64`。
- 扫描版 PDF OCR、PDF 图片/表格理解、MOBI 内嵌图片和自动双页仍未支持。
- Alpha.3 → Alpha.4 App 内自动更新与干净 macOS 环境回归需在候选包发布后完成人工验收。

## [0.2.0-alpha.1] - 2026-07-10

### Added

- Tauri macOS 桌面 App、本地 SQLite 与 App 数据目录存储。
- PDF/MOBI 书库、阅读进度、笔记、聊天、导读缓存和目录式备份。
- 备份预览、校验报告、合并/覆盖导入、sha256 和失败回滚。
- 系统 Keychain API Key 存储、Rust AI transport、请求取消、预算保护和调用诊断。
- 本地健康检查、脱敏诊断包、CI、QA fixtures 和 macOS 签名/公证工具链。
- SemVer 版本同步、校验和升版脚本。
- 设置页和诊断面板中的 App version、发布通道、Git commit/dirty、SQLite schema 与备份格式展示及复制入口。
- Tag 驱动的 macOS GitHub Release 流水线，连接 Developer ID 签名、Apple 公证、staple、Gatekeeper、manifest/checksum 和 artifact 上传。

### Changed

- 测试与正式桌面环境使用独立 bundle identifier、SQLite/files/backups/logs 目录和 Keychain service。
- `npm run tauri:dev` 固定进入 `com.duban.reader.test`，正式环境只能由 formal 配置启动。
- 正式打包同时保留 `.app` 和 `.dmg`，并在公证前验证两者签名。

### Fixed

- 修复连续 AI 请求可能重复触发系统 Keychain 密码弹窗的问题。
- 修复 macOS `asset://` 状态 `0` 导致历史 PDF 无法在 PDF.js 打开的问题；当前等待桌面人工回归。
- 修复历史开发数据误写入正式 App 数据目录的问题，并完成测试数据归位与回滚快照。

### Security

- 增加 CSP、安全头、Tauri command 输入校验、路径边界和敏感信息扫描。
- 备份、日志、诊断和错误详情不包含 API Key、书籍正文或私密内容全文。

### Known limitations

- `v0.2.0-alpha.1` 的首次 tag workflow 在签名前因 runner 未保留 annotated tag object 而停止，没有生成安装包或 GitHub Release。
- 当前正式 macOS 构建仅验证 Apple Silicon `arm64`。
- 自动更新、完整升级 fixtures、压缩备份归档和备份签名仍待后续阶段。
