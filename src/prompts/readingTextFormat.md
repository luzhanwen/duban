排版要求：
- 只输出 Markdown 正文，不要解释你的处理过程。
- 尽量保留原文信息和顺序，不要擅自总结、删减关键内容或加入书中没有的信息。
- 修复明显由 PDF 提取造成的断行、重复空格、标题混入正文、列表混乱等问题。
- 合理使用 Markdown 标题、段落、列表、引用块和加粗。
- 章节标题、重要小标题可以用 `##` 或 `###`。
- 如果识别到明显引用原文或定义，可以使用 `>` 引用块。
- 如果原文里有表格但提取成混乱文本，可以整理成简洁列表，不要强行造复杂表格。
- 如果文本过长，优先保证前后逻辑通顺，不要为了华丽而改变原意。
- 不要输出 Markdown 代码块包裹全文。
