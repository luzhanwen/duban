## 2026-06-08 设置页增加 AI 批量配置

### 需求背景

- 现有设置页需要逐项填写供应商、模型、Base URL、API Key 和价格，调试多个模型配置时不够方便。
- 用户希望应用可以读取一份文档来批量配置 AI 参数，并提供可复用的 TXT 模板。
- 第一版模板仍需要自己填写或调整较多字段，实际使用时不够省心。

### 本次更新

- 设置页新增「AI 批量配置」区域，支持上传 TXT 配置文档。
- 上传后会解析 `key = value` 或 `key: value` 格式，并立即合并保存到本地 IndexedDB。
- 新增「下载模板」入口，模板文件为 `public/ai-config-template.txt`。
- 模板预填 Anthropic、OpenAI、DeepSeek、Kimi 的常用 Base URL、模型名和可用价格字段，用户主要只需要粘贴 API Key。
- TXT 支持 `[anthropic]`、`[openai]`、`[deepseek]`、`[kimi]`、`[openai-compatible]` 分组，也支持完整 key，例如 `openaiCompatible.baseUrl`。
- 只填一个 OpenAI-compatible 供应商的 API Key 时，会自动选择该供应商并写入对应 Base URL、模型和价格。
- 导入时只覆盖文档中填写了值的字段；空值、不认识的字段和格式错误的行会跳过。
- 新增「下载当前配置」按钮，可把当前表单里的 AI 配置导出为可重新导入的 TXT。

### 兼容与限制

- 该功能仍是纯前端 BYOK，不引入后端，也不会把 API Key 上传到项目服务器。
- 当前是批量导入到现有“单个默认供应商 + 两类供应商配置”的设置结构，不是多套模型 profile 管理；如果同时填写多个 OpenAI-compatible API Key，需要用 `provider` 指定默认供应商。
- 导出的当前配置会包含 API Key，只适合保存在用户可信的位置。

## 2026-06-08 增加 MOBI 书籍导入与文本阅读

### 需求背景

- 当前书架只支持 PDF 上传，用户希望可以导入 MOBI 格式书籍。
- 现有阅读器以 PDF 原版页渲染为主；MOBI 没有可直接复用的 PDF canvas/text layer，需要走文本阅读路径。
- 书籍信息、阅读计划、进度、笔记和问答都依赖页码字段，需要兼容 MOBI 的文本分页。

### 本次更新

- 新增 `src/lib/bookFormats.js`，统一识别 PDF/MOBI 文件格式、上传 accept 配置和页码单位展示。
- 新增 `src/lib/mobi.js`，接入 `@lingo-reader/mobi-parser`，支持在浏览器端解析 MOBI/KF8，读取 metadata、toc/spine 和章节 HTML。
- MOBI 正文会清洗为可阅读文本，并按约 2200 字符切成文本页，继续复用现有章节、阅读计划、当前页上下文和笔记数据结构。
- 新增 `src/components/TextBookReader.jsx`，为 MOBI 提供文本阅读视图，支持当前文本页识别、恢复上次位置、划选原文后问读伴或添加笔记。
- 书架上传入口改为支持 PDF 或 MOBI；保存逻辑从 `createBookFromPdf` 扩展为通用 `createBookFromParsedFile`，同时保留旧函数兼容。
- 阅读器根据书籍格式分流：PDF 继续使用 `PdfReader` 原版页渲染，MOBI 使用 `TextBookReader` 文本阅读。
- 书架、书籍信息页、开书计划页、阅读目录、聊天提示和笔记详情中的页码文案改为按格式显示“页”或“文本页”。
- 删除书籍确认文案改为“原始文件”，避免只描述 PDF。

### 兼容与限制

- 已有 PDF 书籍如果没有 `format` 字段，会按文件名回推，默认仍作为 PDF 处理。
- MOBI 当前提供文本阅读和伴读能力，不渲染 Kindle 原版版式，也不会显示 MOBI 内嵌图片。
- MOBI 章节优先使用 TOC 映射；TOC 不完整时按 spine 顺序生成章节，用户仍可在书籍信息页手动调整。
- `@lingo-reader/mobi-parser` 作为动态导入依赖，只在上传 MOBI 时加载，避免直接增加首屏主包。

## 2026-06-08 书架页小清新改版

### 需求背景

- 书架页需要更像真实书架，气质要贴近「读伴」手写 logo 的文艺、小清新感觉。
- 书籍操作不应全部平铺在卡片底部，适合收进菜单或下拉框。
- 书籍需要支持删除，并明确清理本地保存的 PDF、阅读计划、进度、笔记和问答记录。
- 第一版书架仍偏厚重，顶部「读伴书架」banner 信息重复，详情区域占用空间太多。
- 书籍详情适合改成鼠标悬浮或键盘聚焦后展开，让默认书架更轻、更安静。

### 本次更新

- 移除大尺寸书架 banner，改为轻量页面标题栏，上传入口保留在右侧。
- 开发测试书入口从独立横幅移到标题栏按钮区，减少页面冗余。
- 书籍卡片默认只展示书脊、状态、标题、作者、阅读概览和主操作按钮。
- 页数、章节、识别方式、章节角色统计和完整阅读进度改为 hover/focus 展开。
- 书籍次级操作统一放入右上角菜单：阅读目录、书籍信息、开书设置、删除书籍。
- 书脊与按钮使用更柔和的鼠尾草绿、浅蓝、浅紫、浅棕等颜色，降低木质层板的厚重感。
- 删除书籍时增加确认提示，并清理当前书籍相关的本地存储数据。

### 后续修正

- 进一步移除标题区域的面板感，不再显示 Local Library、大背景或明显分隔线。
- 书卡详情展开从纯 CSS hover 改为单本书状态控制，避免鼠标悬浮时多本书同时展开。
- 书架网格改为顶部对齐，避免同一行卡片被展开卡片撑高。
- 进一步执行 Flat 2.0 方向：移除木质托盘和立体书脊，书籍视觉改为 2D 平面封面。
- 定义偏灰紫蓝的全局 Primary，仅用于书卡底部的核心阅读按钮；上传和测试导入改为次级线框按钮。
- 放大数据面板内边距，增强标题、作者、标签和数据之间的字号/字重层级。
- 统一书卡默认高度，避免长短标题导致卡片尺寸参差；悬浮展开时仅当前卡片高度自适应。
- 建立克制的书卷感字体栈，整体靠近 logo 气质，但不使用花哨手写字体。
- 将主色从冷灰紫回调为温暖焦糖棕，封面、状态、进度等辅助色统一到暖杏、橄榄、陶土色系。
- 固定书卡顶部封面/标题信息区高度，让阅读概览与底部主按钮在多卡片之间保持对齐。
- 撤掉小封面装饰方案，回到以书名、作者、状态、页数为核心的信息卡；桌面恢复两列宽卡比例。
- 页面最大宽度收回到 `max-w-5xl`，避免卡片在宽屏中过窄或漂浮。
- 去掉展开详情里的第二条阅读进度条，只保留默认态的阅读概览，展开后展示补充数据和打卡信息。
- 展开动画改为固定卡片高度过渡 + 详情区域 grid 行展开，减少 `max-height` 动画造成的顿挫。

### 设计原则

- 页面顶部只做导航和操作，不再重复品牌表达。
- 默认状态优先呈现书本本身，详情只在用户表达兴趣时展开。
- 操作入口保持克制，但删除等破坏性操作必须可发现、可确认。
- 桌面端用 hover 展开，移动端因为没有稳定 hover，详情保持展开，避免信息不可达。
