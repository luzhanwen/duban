## 2026-07-09 整理这本书前端原型

### 需求背景

- 用户希望在本书级聊天之外，插入一个更像“整本书复盘 / 整理区”的阶段，用来整理全书笔记、重点和读伴回答。
- 这个空间应延续「每本书一个专家读伴」的体验，但不应一开始就做成泛化的全局知识库。
- 页面视觉需要遵守现有「现代数字书斋」标准：米纸质感、淡墨文字、朱砂少量点缀、直接进入可用工作台，不做大面积 banner。

### 本次更新

- 新增 `BookSalon` 页面，作为单本书的「整理这本书」前端原型。
- 书架书籍菜单新增「整理这本书」入口；本书读伴聊天页顶部也可切入整理页。
- 页面采用“左侧本书状态 + 右侧主案面”的双栏结构：
  - 左侧只保留本书概览、当前主线和阅读进度，避免把所有模块同时摊开。
  - 右侧主案面使用「笔记 / 重点 / 复盘」标签页，默认聚焦笔记整理。
  - 重点和复盘不再常驻右栏，而是在同一主案面内切换查看。
- 笔记整理区支持按全部、手记、摘录、读伴筛选，并可直接编辑、保存或二次确认删除笔记。
- 重点卡片的标签改为更直白的读者语言，例如「作者想解释」「带着读的问题」「重要线索」「容易卡住」，避免使用「问题意识」「核心追问」这类偏内部或学术化的词。
- 页面名称从旧入口名收敛为「整理这本书」，避免入口文案绕远或像内部概念。
- 顶部页签从右上角胶囊按钮改为标题下方的轻量下划线切换；笔记筛选从大胶囊组改为左对齐小筛选片，减少首屏右上角的拥挤和装饰感。
- 整理页真实读取本地书籍、阅读进度、全书笔记、本书聊天和读后交流数据；本轮不新增 AI 调用。
- 页面样式按 UI 标准使用米纸表面、柔和纸边、淡墨层级、朱砂进度与关键按钮，并补充桌面/平板/移动端响应式布局。

### 后续修正

- 根据试用反馈收敛第一版三栏布局：去掉右侧常驻知识卡/复盘栏，改为标签页切换，让首屏更像一本书的整理案面，而不是信息面板堆叠。
- 根据后续反馈继续收敛命名和控件：入口使用「整理这本书」，主视图标签使用「笔记 / 重点 / 复盘」，不再使用旧入口名和旧复盘标签这类需要解释的词。
- 根据会客厅首版试用反馈继续收敛视觉与易用性：页面顶部改为「本书会客厅」书案入口，加入朱砂进度小章和更清楚的「和读伴聊聊 / 继续阅读 / 设定读伴」动作；笔记区从后台式列表改为「书笺」列 + 整理区，筛选片、面板边框和知识/复盘卡片都改为更轻的米纸分隔，避免看起来像管理后台或卡片堆叠。
- 根据“内容刚好填满一页”的反馈继续调整会客厅缩放：桌面端改为固定一屏高度，外层不再自然撑出页面滚动；整体宽度从 1480 收到 1320，左右栏比例改为约 42/58，左侧案头卡片和右侧整理区共同填满当前窗口，较矮窗口下自动压缩标题、按钮和编辑区间距。
- 根据“废话太多、不清晰、占地方”的反馈继续裁剪会客厅文案：移除顶部解释句和主案面说明句，把「把书里留下的东西拢在一处」改为「笔记整理」，把「案头概览 / 书笺 / 读到哪里」改为更直接的「本书状态 / 笔记列表 / 当前进度」，减少装饰性语言。

### 兼容与限制

- 本轮是前端页面和本地数据聚合原型，不新增 SQLite schema。
- 重点和复盘当前是基于已有导读与笔记的轻量整理视图，还没有接入 AI 自动归纳、标签体系、跨书关联或导出。
- 笔记编辑复用现有 `book:{id}:notes` / `notes` 数据结构，删除笔记需要二次确认。

## 2026-07-08 本书读伴聊天接入 AI

### 需求背景

- 用户希望不在阅读器里时，也能围绕一本书和读伴聊天，用来思考、回味、接上思路或准备下一次阅读。
- 当前书架菜单已经预留「和读伴聊聊」入口，但页面只是前端占位，本地消息不持久，也没有真正调用模型。
- 产品方向正在强化“每本书配一个专家”的体验，本书聊天应成为每本书的长期读伴空间，而不是 app 里的泛聊天入口。

### 本次更新

- `BookCompanionChat` 页面从占位回复升级为真实本书级聊天：
  - 首次进入加载本书级历史消息。
  - 发送后使用流式 AI 回答。
  - 支持 Enter 发送、Shift+Enter 换行、生成中停止、错误提示和自动滚动。
  - 回答完成后展示模型、输入 token、输出 token、估算费用和输出上限提示。
- 新增 `bookCompanionChat.md` prompt 和 `bookCompanionChat.js` 业务层：
  - 上下文带入书名作者、读伴记忆、整本书地图、阅读计划、当前进度、当前阅读项摘录、最近笔记、阅读中问答和读后交流。
  - 默认不把整本书全文塞进模型，只放当前阅读位置附近的少量摘录。
  - 用户问到未读后文时，读伴应先提醒可能剧透，并优先给不剧透回答。
- 本书聊天复用现有 `book:{id}:chat` 数据通道，用保留 key `__book_companion__` 区分全书聊天，不影响阅读器 sidebar 按阅读项保存的伴读问答。
- 本书读伴聊天 P2 新增沉淀和管理动作：
  - 每条读伴回答可一键「记到笔记」，优先保存到当前阅读项笔记；没有可用阅读项时，保存到本书读伴保留分组。
  - 已保存的回答会显示「已记到笔记」，重新进入页面后会按 `sourceMessageId` 识别，避免重复保存。
  - 顶部新增「清空聊天」二次确认，只清空本书聊天历史，不删除书籍、阅读进度或已保存笔记。
- 设置页任务模型 Profile、AI 预算和诊断标签新增「本书读伴聊天」任务类型。

### 兼容与限制

- 当前 P0-P2 先解决真实聊天、全书上下文、进度上下文、本地持久化、回答记到笔记和聊天清空；还没有做回味卡片、长期记忆自动摘要或聊天导出入口。
- 本书聊天会把必要的阅读进度、读伴记忆、当前阅读项摘录和用户沉淀发送给当前模型服务商；隐私说明已同步。
- 没有修改 SQLite schema；聊天历史继续复用 `chat_messages`，从本书聊天保存的回答继续复用 `notes`，并用 `source = "book-companion-chat"` 标记来源。

### 后续修正

- 修复书架菜单「和读伴聊聊」点击无反应：`Shelf` 渲染 `BookCard` 时漏传 `onChatBook`，导致菜单项内部拿不到 App 级视图切换函数；现在已补上传参，并在卡片内部加了可选调用兜底。
- 修复本书读伴聊天页在较矮桌面窗口里底部输入区被挤出视口的问题：桌面宽度下页面改为按当前 `100dvh` 高度布局，左侧信息栏和右侧消息区各自内部滚动，输入框固定留在聊天面板底部；窄屏仍保持自然纵向滚动。
- 左侧信息栏同步适配当前窗口高度：长书名、作者和当前阅读位置标题限行换行，桌面端第三块“读伴会参考”改为内部滚动，矮窗口下压缩头像、字号、卡片间距和进度区，让三块信息不会把页面整体撑出视口。

## 2026-07-08 视觉标准 V1.0 与全局底座

### 需求背景

- 用户确认新的主视觉方向：米纸质感、朱砂强调、淡墨文字、藏书章图形、柔和边框，以及更安静的东方文人气。
- 旧版全局视觉仍混有焦糖棕强调、偏普通纸卡和局部硬编码色值，需要先建立统一的底层标准，再分页面迁移。
- 当前阶段先做部署步骤 1/2/3：项目视觉标准、Design Tokens 和全局背景/导航底座，不直接重做书卡结构。

### 本次更新

- `UI_DESIGN_STANDARDS.md` 升级为「现代数字书斋」方向，明确米纸、朱砂猫章、现代藏书票、淡墨文字和猫伴读的视觉边界。
- 视觉标准从焦糖棕主色切换为 V1.0 token：米纸背景、纸面表面、柔和边框、淡墨文字、朱砂强调和朱砂 hover。
- Tailwind 主题色同步接入 V1.0 token，并保留现有 `paper`、`paper-card`、`accent`、`line` 等 class 名，降低后续迁移风险。
- 全局 CSS 新增同名 CSS variables，让旧的 `--app-primary`、`--surface-soft`、`--font-literary-ui` 等变量映射到新标准。
- 全局背景改为米纸底色加固定纸纹氛围层，纸纹以低透明度呈现，避免压过小字号文字。
- 顶部导航切到半透明米纸表面，当前页增加克制朱砂短下划线；书架入口文案同步为「藏书」。

### 后续修正

- 保留原本全屏开屏动画体验：修正全局纸纹层级规则，避免 `.app-root` 直接子元素规则覆盖开屏的 `fixed` 定位；开屏层显式保持固定全屏遮罩。
- 修复刷新时先闪出空书架再出现开屏动画的问题：开屏遮罩首帧保持不透明，书架在本地数据读取完成前显示「正在整理藏书」，不再用空状态占位。

### 兼容与限制

- 本次还没有重做书架书卡结构，藏书票式书卡会在后续步骤单独推进。
- 开书页、阅读器和设置页里仍有部分旧硬编码色值，后续会按主流程逐步替换。
- 标准文档提到的参考资产目录当前未进入项目，本轮先复用现有 `paper-texture.png` 和 `logo.svg`。

## 2026-07-08 书架卡片改为藏书票纸卡

### 需求背景

- 用户希望书架卡片接近参考图中的纸质藏书票感觉：米纸卡片、嵌入式封面、书名登记、朱砂小章和细进度线。
- 旧书架以封面为主体，封面下方只显示一行状态，私藏感和纸本登记感还不够明确。

### 本次更新

- 书卡外层改为米纸纸卡，加入柔和边框、纸张阴影和轻微纸纹层。
- 封面区域保留原有 PDF 真实封面和 MOBI/失败文字封面兜底，但从裸封面改为嵌在纸卡上方的内嵌封面。
- 书卡下半部分新增登记区：书名、作者、虚线分隔、`读伴藏书 · 第001卷`、阅读状态和朱砂细进度线。
- 右下角加入朱砂小章，菜单按钮收进登记区右上角。
- 阅读状态文案改为纸本语气，例如「未启卷」「已读 2% · 今日翻阅」「已读完 · 可回顾」。
- 书架网格列宽加宽，移动端维持双列，并压缩标题、登记信息和朱砂章尺寸。
- 文字封面兜底色从多彩封面收敛为淡墨、朱砂、纸灰和低饱和青灰系。

### 后续修正

- 右下角小章从纯装饰猫章改为语义状态章，优先显示「已读完」，未完成书籍按当天记录显示「今日已读」或「今日未读」。
- 未确认书目信息或未完成读伴设定的书籍保留待处理章，避免未进入阅读流程的书被误标为今日状态。
- 登记区阅读文案收敛为进度信息，不再和状态章重复显示「今日翻阅」；0% 进度时隐藏朱砂进度点，避免「未启卷」与已开始的视觉暗示冲突。
- 书架待处理状态继续收敛为用户能直接理解的「未设置读伴」：未确认书籍信息时补充「先确认书籍信息」，已确认但未完成计划时补充「还未定下陪读方式」；主操作统一为「设定读伴」。
- 术语继续校准：将偏游戏语境的「捏读伴 / 捏专属读伴 / 捏伴」统一改为「设定读伴」，让陌生用户也能理解这是在为当前书确定陪读方式。
- AI 封面生成先搁置，优先调整现有封面渲染质感：保留轻微 3D 倾斜和抬升，但移除鼠标跟随亮斑、斜向扫光、强 hover 阴影和增饱和效果，改为纸纹覆盖、书脊压痕和哑光边缘。
- 继续收敛用户可见入口文案：书籍信息页、书架菜单、阅读器空计划提示和读伴设定页里的「开书设置 / 开书分析 / 开书地图」改为「设定读伴 / 读伴地图 / 读伴小纸条」，避免选项层面混用术语。
- 修复哑光纸面书封 hover 时封面被纸纹/信息层压暗到近似消失的问题：纸纹层降为轻透明表面质感，信息层只保留底部渐变，确保真实封面始终可见。
- 书卡已有完整登记区后，封面 hover 不再重复显示状态、页数、进度和章节，只保留主操作提示；「未设置读伴」状态文案去掉后缀说明，避免长文本撑乱书卡对齐。
- 为每本书预留独立读伴聊天入口：书卡菜单新增「和读伴聊聊」，进入单独的本书读伴聊天页面；当前先完成前端结构，展示书籍信息、阅读位置、读伴上下文、建议问题和聊天输入区，后续再接入本书级长期聊天与模型调用。

### 兼容与限制

- 封面主点击、hover overlay、右键菜单和省略号菜单逻辑保持不变。
- 本轮只改书架卡片视觉结构，没有改阅读目录弹窗和阅读器里的目录列表样式。

## 2026-07-06 开书设置改为读伴设定式对话

### 需求背景

- 旧版开书设置把“生成整本书导读”放在第一步，等待时间长，用户容易被卡住。
- 生成完成后页面仍展示大量分析、路线、难点和消耗信息，视觉上像任务还没结束。
- 快速导读容易抽象化，甚至用“这本书不是……”开头，先纠正用户而不是邀请用户进入书。
- 用户想先告诉读伴自己为什么读、以前知道什么、对哪里感兴趣；这更像开书前的一段相识，而不是后台配置。

### 本次更新

- 开书步骤第一步从「开书地图」改为「设定读伴」。

## 2026-06-09 导读页动效优化

### 需求背景

- 书架首页加入封面入场、hover 抬升和鼠标跟随旋转后，进入导读页会显得明显静态。
- 导读页承担“从书架进入阅读”的过渡，需要有动感，但仍要保持安静、克制、适合读书前准备。

### 本次更新

- 导读页增加分层入场：顶部操作、章节标题、开场卡片和底部按钮按顺序轻量浮入。
- 开场卡片加入细线扫入和更柔和的纸面阴影，延续首页的高级感但不做拟物化。
- 导读生成前的提示卡、生成中的 loading 卡和生成后的 overview/list 卡片统一动效语言。
- 导读目标和读前问题改为逐条错峰出现，并在 hover/focus 时有轻微位移与阴影反馈。
- 生成中状态增加进度条呼吸、骨架流光和步骤错峰出现，让用户感知系统正在整理内容。
- 阅读页侧栏里的“提示”面板同步使用导读动效，避免从导读页进入正文后突然变静态。
- 点击「翻开这一章」时增加整页翻页过渡，让导读页像一整张纸翻走，再露出正文页。
- 所有导读动效纳入 `prefers-reduced-motion` 兜底。

### 后续修正

- 2026-06-10：翻页过渡从硬翻 180 度改为短弧线翻动，并加入移动折痕、高光和纸页阴影；翻页期间底层页面改为瞬时定位，避免过渡时出现额外滚动感。
- 2026-06-10：优化翻页性能：阅读页改为在翻页开始时挂载，避免动画中段重渲染；折痕、高光和阴影改为 transform/opacity 驱动，移除容易掉帧的布局动画和滤镜。
- 2026-06-10：进一步提升翻页帧率感：取消全屏 3D 旋转，改为更轻的 2D 纸面滑翻；减少运动层和关键帧停顿点，优先保证连续、稳定的 60fps 观感。
- 2026-06-11：去掉翻页 sheet 和折痕的多段中间关键帧，改为单条连续曲线扫出，避免视觉上出现“起、翻、收”三拍。
- 2026-06-11：翻页时长从 640ms 微调到 720ms，保留单曲线扫出的连续感，同时让过渡不再显得过快。
- 2026-06-12：翻页时长调至 1040ms，并将纸面位移与透明度拆分；纸面在大部分过渡中保持可见，最后自然淡出，增强优雅翻书过程感。
- 2026-06-12：翻页开场增加短暂停留，先让章节名保持可读，再进入翻页动作，避免用户还没看清标题就被带走。
- 2026-06-15：翻页总时长调至 1460ms，阅读页在过渡开始时即挂载，纸面保留更长时间后淡出；活动记录延后到过渡结束，避免翻页过程中滚动、记录和渲染状态互相打架。
- 2026-06-15：翻页层进一步改成轻量 2D 纸面滑翻：移除背面 3D 翻转，折痕、高光、阴影全部使用 `transform/opacity`，并用 `contain: paint` 限制重绘范围。

### 设计原则

- 动效服务“进入阅读”的节奏，不抢正文和导读内容的注意力。
- 页面只做 2D 平面动效，不加入立体拟物、装饰光球或大面积渐变背景。
- hover 动效只表达可交互和信息层级，不改变核心布局和阅读顺序。

## 2026-06-09 修复每日阅读日期判断

### 需求背景

- 用户昨天完成一个阅读项后，今天回到书架仍看到「提前开始下一章阅读」。
- 旧逻辑只知道“当前阅读项已完成”，不知道下一阅读项的计划日期是否已经到了今天。

### 本次更新

- 书架会比较下一阅读项的计划日期：
  - 下一项计划日已经到达时，主按钮显示「开始今日阅读」。
  - 下一项计划日还没到时，主按钮才显示「提前开始下一章阅读」。
- 今日完成页也按下一项计划日调整按钮文案：
  - 计划日已到：显示「开始下一项阅读」。
  - 计划日未到：显示「提前开始下一章阅读」。
- 完成阅读时记录每个阅读项的完成时间，后续可以更准确地区分“昨天完成”和“今天完成”。
- 书架顶部新增「今天日期」和全局「上次阅读」。
- 书卡摘要和展开详情保留上次阅读日期。
- 计划日期不再显示在书卡、阅读目录、阅读器开场页、正文页、读后交流页和今日完成页，避免误解它到底指当前项还是下一项。

### 兼容与限制

- 旧进度数据没有每项完成时间，默认按空对象兼容，不会影响已完成项。
- 是否“提前”仍会在内部依据阅读计划项里的 `date` 字段判断；没有日期的旧计划项会被视为已经到达计划日。

## 2026-06-08 设置页增加 AI 批量配置

### 需求背景

- 现有设置页需要逐项填写供应商、模型、Base URL、API Key 和价格，调试多个模型配置时不够方便。
- 用户希望应用可以读取一份文档来批量配置 AI 参数，并提供可复用的 TXT 模板。

- 新增「下载模板」入口，模板文件为 `public/ai-config-template.txt`。

- 当前仍是 BYOK 配置导入，不导入多套 API Key；P6.4.6 起支持 `[profiles]` 导入任务级非敏感模型 profile，如果同时填写多个 OpenAI-compatible API Key，仍需要用 `provider` 指定默认供应商。
- 导出的当前配置会包含 API Key，只适合保存在用户可信的位置。
