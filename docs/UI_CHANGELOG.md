## 2026-06-15 阅读器新增本书读伴记忆面板

### 需求背景

- 开书契约上下文已经接入章节导读、阅读中问答和读后交流，用户需要在阅读过程中随时查看和调整单本书的陪读方向。
- 第一版记忆只服务当前书籍，不做全局用户记忆，也不改历史聊天、导读缓存或提示词结构。

### 本次更新

- 在阅读器右侧侧栏面板切换中新增「记忆」，与「问读伴 / 提示 / 笔记 / 阅读项」同级。
- 面板标题为「本书读伴记忆」，默认读取 `book.readingProfile.companionFocus`。
- 旧书没有 `companionFocus` 时，使用开书契约上下文的 fallback 记忆，避免空白或崩溃。
- 支持编辑用户阅读目标、陪读重点类型、读伴当前理解和后续陪读规则。
- 保存按钮仅在内容变更后启用；保存成功后保留在当前阅读页，不跳转、不打断阅读节奏。
- 保存成功会更新当前阅读器里的本地 book state，让后续导读、问答和读后交流读取新的陪读方向。
- 视觉上沿用阅读器侧栏结构，只使用表单、轻量提示和克制按钮，不新增大面积说明卡。

### 兼容与限制

- 该面板只修改当前书籍的读伴记忆，不改全局记忆。
- 不修改 AI prompt、`readingContract.js` 核心逻辑、历史聊天记录、已生成导读缓存或 wholeBookGuide。
- 当前使用普通 textarea 和 select，暂不引入富文本编辑器。

- `src/lib/books.js`

## 2026-06-09 书架改为封面优先测试版

### 需求背景

- 用户希望效仿 Apple Books 的书架表现：默认只展示书籍封面，让书本本身成为视觉主体。
- 旧版虽然信息完整，但页面标题、解释文本、卡片边框和封面下方标题会一起抢占注意力，和 Apple Books 的轻量书架相比显得复杂。

### 本次更新

- 书架标题区收敛为「全部」和轻量元信息，移除大段说明。
- 书籍从宽信息卡改为封面网格，内容区宽度放大，更接近桌面书架视图。
- PDF 书籍优先读取原始文件第一页，渲染并缓存为封面缩略图。
- MOBI 或封面渲染失败时，使用带书名和作者的 2D 文字封面兜底。
- 封面本身作为主操作入口：未确认进入书籍信息，未规划进入开书设置，已规划进入阅读。
- 次级操作收敛到封面下方的小省略号菜单，不再压在封面右上角。
- 封面下方只保留轻量状态或进度，例如「已规划」「15%」「已读完」，弱化标题和作者外显。
- 统一顶部导航和书架内容的最大宽度与左右内边距，避免 logo、标题和书本起点错位。
- 收紧标题到封面行的垂直距离、封面网格列宽和横向间距，让书架比例更接近 Apple Books。
- 底部书籍统计改为跟随页面底部节奏，不再悬在首屏中间。
- 增加克制动效：页面和标题轻入场、封面错峰升起、封面 hover 轻微抬升和缩放、信息 overlay 渐进出现。
- 为封面动效保留 `prefers-reduced-motion` 兜底，避免动效成为使用前提。
- 修复入场动画占用 `transform` 导致 hover 位移不明显的问题；将封面淡入与封面落位拆成两层动画。
- 加强 hover 反馈：封面抬升幅度、缩放和阴影略增强，并加入轻微光泽扫过效果。
- 增加展示柜式鼠标跟随效果：鼠标在封面上移动时，封面会按指针位置轻微 `rotateX/rotateY`，并带有跟随鼠标的柔光点。

### 兼容与限制

- 当前是测试性改动，未引入 Apple Books 式侧边栏。
- PDF 封面来自第一页，如果第一页本身是内封、空白页或 3D 书影，书架会忠实显示该页。
- MOBI 暂无真实封面提取能力，仍使用文字封面。
- 鼠标跟随旋转只在非触控指针和非 reduced-motion 环境启用。

## 2026-06-09 导读页动效优化

### 需求背景

- 书架首页加入封面入场、hover 抬升和鼠标跟随旋转后，进入导读页会显得明显静态。
- 导读页承担“从书架进入阅读”的过渡，需要有动感，但仍要保持安静、克制、适合读书前准备。

### 本次更新

- 导读页增加分层入场：顶部操作、章节标题、开场卡片和底部按钮按顺序轻量浮入。
- 开场卡片加入细线扫入和更柔和的纸面阴影，延续首页的高级感但不做拟物化。
- 导读生成前的提示卡、生成中的 loading 卡和生成后的 overview/list 卡片统一动效语言。
- 导读目标和读前问题改为逐条错峰出现，并在 hover/focus 时有轻微位移与阴影反馈。
- 生成中状态增加进度条呼吸、骨架流光和步骤错峰出现，让用户感知系统正在整理内容。
- 阅读页侧栏里的“提示”面板同步使用导读动效，避免从导读页进入正文后突然变静态。
- 点击「翻开这一章」时增加整页翻页过渡，让导读页像一整张纸翻走，再露出正文页。
- 所有导读动效纳入 `prefers-reduced-motion` 兜底。

### 后续修正

- 2026-06-10：翻页过渡从硬翻 180 度改为短弧线翻动，并加入移动折痕、高光和纸页阴影；翻页期间底层页面改为瞬时定位，避免过渡时出现额外滚动感。
- 2026-06-10：优化翻页性能：阅读页改为在翻页开始时挂载，避免动画中段重渲染；折痕、高光和阴影改为 transform/opacity 驱动，移除容易掉帧的布局动画和滤镜。
- 2026-06-10：进一步提升翻页帧率感：取消全屏 3D 旋转，改为更轻的 2D 纸面滑翻；减少运动层和关键帧停顿点，优先保证连续、稳定的 60fps 观感。
- 2026-06-11：去掉翻页 sheet 和折痕的多段中间关键帧，改为单条连续曲线扫出，避免视觉上出现“起、翻、收”三拍。
- 2026-06-11：翻页时长从 640ms 微调到 720ms，保留单曲线扫出的连续感，同时让过渡不再显得过快。
- 2026-06-12：翻页时长调至 1040ms，并将纸面位移与透明度拆分；纸面在大部分过渡中保持可见，最后自然淡出，增强优雅翻书过程感。
- 2026-06-12：翻页开场增加短暂停留，先让章节名保持可读，再进入翻页动作，避免用户还没看清标题就被带走。
- 2026-06-15：翻页总时长调至 1460ms，阅读页在过渡开始时即挂载，纸面保留更长时间后淡出；活动记录延后到过渡结束，避免翻页过程中滚动、记录和渲染状态互相打架。
- 2026-06-15：翻页层进一步改成轻量 2D 纸面滑翻：移除背面 3D 翻转，折痕、高光、阴影全部使用 `transform/opacity`，并用 `contain: paint` 限制重绘范围。

### 设计原则

- 动效服务“进入阅读”的节奏，不抢正文和导读内容的注意力。
- 页面只做 2D 平面动效，不加入立体拟物、装饰光球或大面积渐变背景。
- hover 动效只表达可交互和信息层级，不改变核心布局和阅读顺序。

## 2026-06-09 修复每日阅读日期判断

### 需求背景

- 用户昨天完成一个阅读项后，今天回到书架仍看到「提前开始下一章阅读」。
- 旧逻辑只知道“当前阅读项已完成”，不知道下一阅读项的计划日期是否已经到了今天。

### 本次更新

- 书架会比较下一阅读项的计划日期：
  - 下一项计划日已经到达时，主按钮显示「开始今日阅读」。
  - 下一项计划日还没到时，主按钮才显示「提前开始下一章阅读」。
- 今日完成页也按下一项计划日调整按钮文案：
  - 计划日已到：显示「开始下一项阅读」。
  - 计划日未到：显示「提前开始下一章阅读」。
- 完成阅读时记录每个阅读项的完成时间，后续可以更准确地区分“昨天完成”和“今天完成”。
- 书架顶部新增「今天日期」和全局「上次阅读」。
- 书卡摘要和展开详情保留上次阅读日期。
- 计划日期不再显示在书卡、阅读目录、阅读器开场页、正文页、读后交流页和今日完成页，避免误解它到底指当前项还是下一项。

### 兼容与限制

- 旧进度数据没有每项完成时间，默认按空对象兼容，不会影响已完成项。
- 是否“提前”仍会在内部依据阅读计划项里的 `date` 字段判断；没有日期的旧计划项会被视为已经到达计划日。

## 2026-06-08 设置页增加 AI 批量配置

### 需求背景

- 现有设置页需要逐项填写供应商、模型、Base URL、API Key 和价格，调试多个模型配置时不够方便。
- 用户希望应用可以读取一份文档来批量配置 AI 参数，并提供可复用的 TXT 模板。
- 第一版模板仍需要自己填写或调整较多字段，实际使用时不够省心。

### 本次更新

- 设置页新增「AI 批量配置」区域，支持上传 TXT 配置文档。
- 上传后会解析 `key = value` 或 `key: value` 格式，并立即合并保存到本地 IndexedDB。
- 新增「下载模板」入口，模板文件为 `public/ai-config-template.txt`。
- 模板预填 Anthropic、OpenAI、DeepSeek、Kimi 的常用 Base URL、模型名和可用价格字段，用户主要只需要粘贴 API Key。
- TXT 支持 `[anthropic]`、`[openai]`、`[deepseek]`、`[kimi]`、`[openai-compatible]` 分组，也支持完整 key，例如 `openaiCompatible.baseUrl`。
- 只填一个 OpenAI-compatible 供应商的 API Key 时，会自动选择该供应商并写入对应 Base URL、模型和价格。
- 导入时只覆盖文档中填写了值的字段；空值、不认识的字段和格式错误的行会跳过。
- 新增「下载当前配置」按钮，可把当前表单里的 AI 配置导出为可重新导入的 TXT。

### 兼容与限制

- 该功能仍是纯前端 BYOK，不引入后端，也不会把 API Key 上传到项目服务器。
- 当前是批量导入到现有“单个默认供应商 + 两类供应商配置”的设置结构，不是多套模型 profile 管理；如果同时填写多个 OpenAI-compatible API Key，需要用 `provider` 指定默认供应商。
- 导出的当前配置会包含 API Key，只适合保存在用户可信的位置。

- `src/lib/aiConfigImport.js`

## 2026-06-08 增加 MOBI 书籍导入与文本阅读

### 需求背景

- 当前书架只支持 PDF 上传，用户希望可以导入 MOBI 格式书籍。
- 现有阅读器以 PDF 原版页渲染为主；MOBI 没有可直接复用的 PDF canvas/text layer，需要走文本阅读路径。
- 书籍信息、阅读计划、进度、笔记和问答都依赖页码字段，需要兼容 MOBI 的文本分页。

### 本次更新

- 新增 `src/lib/bookFormats.js`，统一识别 PDF/MOBI 文件格式、上传 accept 配置和页码单位展示。
- 新增 `src/lib/mobi.js`，接入 `@lingo-reader/mobi-parser`，支持在浏览器端解析 MOBI/KF8，读取 metadata、toc/spine 和章节 HTML。
- MOBI 正文会清洗为可阅读文本，并按约 2200 字符切成文本页，继续复用现有章节、阅读计划、当前页上下文和笔记数据结构。
- 新增 `src/components/TextBookReader.jsx`，为 MOBI 提供文本阅读视图，支持当前文本页识别、恢复上次位置、划选原文后问读伴或添加笔记。
- 书架上传入口改为支持 PDF 或 MOBI；保存逻辑从 `createBookFromPdf` 扩展为通用 `createBookFromParsedFile`，同时保留旧函数兼容。
- 阅读器根据书籍格式分流：PDF 继续使用 `PdfReader` 原版页渲染，MOBI 使用 `TextBookReader` 文本阅读。
- 书架、书籍信息页、开书计划页、阅读目录、聊天提示和笔记详情中的页码文案改为按格式显示“页”或“文本页”。
- 删除书籍确认文案改为“原始文件”，避免只描述 PDF。

### 兼容与限制

- 已有 PDF 书籍如果没有 `format` 字段，会按文件名回推，默认仍作为 PDF 处理。
- MOBI 当前提供文本阅读和伴读能力，不渲染 Kindle 原版版式，也不会显示 MOBI 内嵌图片。
- MOBI 章节优先使用 TOC 映射；TOC 不完整时按 spine 顺序生成章节，用户仍可在书籍信息页手动调整。
- `@lingo-reader/mobi-parser` 作为动态导入依赖，只在上传 MOBI 时加载，避免直接增加首屏主包。

## 2026-06-08 书架页小清新改版

### 需求背景

- 书架页需要更像真实书架，气质要贴近「读伴」手写 logo 的文艺、小清新感觉。
- 书籍操作不应全部平铺在卡片底部，适合收进菜单或下拉框。
- 书籍需要支持删除，并明确清理本地保存的 PDF、阅读计划、进度、笔记和问答记录。
- 第一版书架仍偏厚重，顶部「读伴书架」banner 信息重复，详情区域占用空间太多。
- 书籍详情适合改成鼠标悬浮或键盘聚焦后展开，让默认书架更轻、更安静。

### 本次更新

- 移除大尺寸书架 banner，改为轻量页面标题栏，上传入口保留在右侧。
- 开发测试书入口从独立横幅移到标题栏按钮区，减少页面冗余。
- 书籍卡片默认只展示书脊、状态、标题、作者、阅读概览和主操作按钮。
- 页数、章节、识别方式、章节角色统计和完整阅读进度改为 hover/focus 展开。
- 书籍次级操作统一放入右上角菜单：阅读目录、书籍信息、开书设置、删除书籍。
- 书脊与按钮使用更柔和的鼠尾草绿、浅蓝、浅紫、浅棕等颜色，降低木质层板的厚重感。
- 删除书籍时增加确认提示，并清理当前书籍相关的本地存储数据。

### 后续修正

- 进一步移除标题区域的面板感，不再显示 Local Library、大背景或明显分隔线。
- 书卡详情展开从纯 CSS hover 改为单本书状态控制，避免鼠标悬浮时多本书同时展开。
- 书架网格改为顶部对齐，避免同一行卡片被展开卡片撑高。
- 进一步执行 Flat 2.0 方向：移除木质托盘和立体书脊，书籍视觉改为 2D 平面封面。
- 定义偏灰紫蓝的全局 Primary，仅用于书卡底部的核心阅读按钮；上传和测试导入改为次级线框按钮。
- 放大数据面板内边距，增强标题、作者、标签和数据之间的字号/字重层级。
- 统一书卡默认高度，避免长短标题导致卡片尺寸参差；悬浮展开时仅当前卡片高度自适应。
- 建立克制的书卷感字体栈，整体靠近 logo 气质，但不使用花哨手写字体。
- 将主色从冷灰紫回调为温暖焦糖棕，封面、状态、进度等辅助色统一到暖杏、橄榄、陶土色系。
- 固定书卡顶部封面/标题信息区高度，让阅读概览与底部主按钮在多卡片之间保持对齐。
- 撤掉小封面装饰方案，回到以书名、作者、状态、页数为核心的信息卡；桌面恢复两列宽卡比例。
- 页面最大宽度收回到 `max-w-5xl`，避免卡片在宽屏中过窄或漂浮。
- 去掉展开详情里的第二条阅读进度条，只保留默认态的阅读概览，展开后展示补充数据和打卡信息。
- 展开动画改为固定卡片高度过渡 + 详情区域 grid 行展开，减少 `max-height` 动画造成的顿挫。
- 书卡展开增加轻量下探提示符，并为 hover 展开加入短暂意图延迟；去掉整卡上浮位移，让展开从“突然弹开”改为更平缓的内容铺开。
- 修复卡片固定高度过小导致底部阅读按钮脱离卡片的问题；按钮改为卡片底部固定操作区，默认态和展开态重新分配高度与底部留白。

### 设计原则

- 页面顶部只做导航和操作，不再重复品牌表达。
- 默认状态优先呈现书本本身，详情只在用户表达兴趣时展开。
- 操作入口保持克制，但删除等破坏性操作必须可发现、可确认。
- 桌面端用 hover 展开，移动端因为没有稳定 hover，详情保持展开，避免信息不可达。
