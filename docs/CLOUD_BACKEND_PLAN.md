# P9 云后端与多设备同步实施计划

> 建立日期：2026-07-13
> 状态：方向已确定，P9 前不进入云端实现

## 定位

P9 承接原 P6.12 的云同步/后端决策，并把它升级为独立产品阶段。云后端服务多设备和可选云能力，不替代桌面端或手机版的本地核心功能。

## 核心原则

- 无账号仍可阅读、记笔记和使用 BYOK。
- 默认不同步 PDF 原文件，用户明确授权后才允许上传。
- API Key 默认保存在设备安全存储，不作为普通同步数据。
- 同步协议先解决离线、冲突和幂等，再追求实时体验。
- 云备份、数据同步和模型代理分别设计、分别授权、分别计费。

## 编号步骤

- **P9.1 需求与数据流 ADR**：明确同步对象、目的、威胁模型、成本和不做事项。
- **P9.2 账号与设备身份**：登录、设备管理、撤销、删除账号和匿名本地模式。
- **P9.3 同步协议**：版本向量/操作日志选择、增量上传、冲突策略、离线队列和幂等。
- **P9.4 加密与密钥恢复**：端到端/服务端加密决策、恢复密钥和设备丢失处理。
- **P9.5 后端基础设施**：API、数据库、对象存储、任务队列、监控、限流和灾备。
- **P9.6 结构化数据同步**：先上线进度、笔记、陪读记录和非敏感设置，再评估原文件。
- **P9.7 可选云服务**：托管备份、模型代理和其他付费能力保持模块化。
- **P9.8 合规与 Beta**：隐私政策、数据导出删除、成本预警、灰度和跨设备 QA。

## 进入条件

- P8 本地手机版能够独立使用，真实证明跨设备同步是用户问题，而不是技术假设。
- P6 的备份、迁移、诊断和发布基线已稳定。
- 开工前必须先更新后端开发标准、隐私政策、数据流图和威胁模型。
