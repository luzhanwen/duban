# Privacy Policy

Last updated: 2026-06-18

读伴是一个本地优先的纯前端 AI 伴读应用。当前版本没有自己的后端账号、云书库、遥测或分析服务。你的书籍、API Key、阅读进度、笔记和聊天记录默认保存在当前浏览器中；当你主动使用 AI 能力时，必要文本会发送给你选择的模型服务商。

## What Is Stored Locally

读伴使用浏览器 IndexedDB 保存本地数据，包括：

- 原始 PDF/MOBI 文件。
- 提取后的分页文本、章节信息和封面缓存。
- 阅读计划、阅读进度、最近页码和打卡日期。
- 高亮、原文摘录、笔记和从 AI 回答保存的笔记。
- 阅读中的伴读问答和读后交流记录。
- 模型供应商、模型名、Base URL、价格配置和 API Key。

这些数据保存在当前浏览器配置文件中。清空浏览器站点数据、换浏览器、换设备或使用隐私模式，都可能导致数据不可用或丢失。

## What Is Sent To AI Providers

读伴只在你主动触发 AI 功能时发送请求，例如测试连接、生成整本书导读、生成章节导读、阅读中问答、选中文字提问或读后交流。

根据具体功能，请求中可能包含：

- 你的 API Key，请求头会发送给当前模型服务商。
- 书名、作者、章节范围和阅读计划信息。
- 当前章节文本、当前页文本、全书抽样文本或你选中的原文。
- 最近几条伴读问答、读后交流历史。
- 当你选择带入上下文时，相关高亮、笔记和 AI 回答摘录。

读伴不会把这些内容发送到读伴自己的服务器。模型服务商如何处理请求数据，以你选择的服务商政策为准。

## API Key And BYOK

读伴采用 BYOK（Bring Your Own Key）模式。API Key 由用户自己提供，并保存在当前浏览器 IndexedDB 中。

需要注意：

- IndexedDB 不是硬件级安全存储。
- 不可信设备、浏览器扩展、同源页面脚本、本机恶意软件或能访问浏览器配置文件的人，可能读取本地数据。
- 下载当前 AI 配置会生成包含 API Key 的 TXT 文件，请只保存在可信位置。
- 建议为读伴使用单独 API Key，并在模型服务商后台设置额度、限额或调用范围。

## Custom Base URL

OpenAI-compatible 配置允许填写自定义 Base URL。测试连接和生成内容时，API Key 与必要阅读文本会发送到该地址。

读伴会对非官方或非 HTTPS 的 Base URL 做二次确认，但无法替你验证第三方服务是否可信。请只在你完全信任该服务商或本地代理时使用自定义地址。

## What Is Not Collected

当前版本不会主动收集：

- 账号、邮箱或身份信息。
- 设备指纹、广告标识或跨站追踪数据。
- 使用统计、遥测或产品分析事件。
- 书籍文件、笔记或聊天记录的云端副本。

## Data Control

你可以在设置页清空全部本地数据。清空后，书籍、进度、导读缓存、聊天记录、笔记和 API Key 都会从读伴的 IndexedDB 中删除，且无法恢复。

后续版本可能加入书库导出/导入能力。在此之前，请不要把浏览器 IndexedDB 视为长期书库的唯一备份。

## Third-Party Services

当你使用 Anthropic、OpenAI、DeepSeek、Kimi 或其他 OpenAI-compatible 服务时，请同时阅读对应服务商的隐私政策、数据使用政策和 API 条款。

读伴无法控制第三方模型服务商如何保存、处理或审查请求数据。
