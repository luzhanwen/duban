# Privacy Policy

Last updated: 2026-07-08

读伴是一个本地优先的 AI 伴读应用。当前版本没有自己的后端账号、云书库、遥测或分析服务。浏览器版数据默认保存在当前浏览器中；桌面版数据默认保存在本机 App 数据目录、SQLite 和系统 Keychain 中。当你主动使用 AI 能力时，必要文本会发送给你选择的模型服务商。

## What Is Stored Locally

浏览器版使用 IndexedDB 保存本地数据，包括：

- 原始 PDF/MOBI 文件。
- 提取后的分页文本、章节信息和封面缓存。
- 阅读计划、阅读进度、最近页码和打卡日期。
- 高亮、原文摘录、笔记和从 AI 回答保存的笔记。
- 阅读中的伴读问答和读后交流记录。
- 模型供应商、模型名、Base URL、价格配置和 API Key。

这些数据保存在当前浏览器配置文件中。清空浏览器站点数据、换浏览器、换设备或使用隐私模式，都可能导致数据不可用或丢失。

桌面版使用 SQLite + App 数据目录文件保存长期书库数据，包括：

- 书籍元数据、章节索引、分页文本、阅读计划、阅读进度、笔记、伴读聊天、读后交流、章节导读和 AI 排版缓存。
- 原始 PDF/MOBI 文件和封面缓存文件。
- 非敏感 AI 设置，例如供应商、模型名、Base URL、价格配置、预算配置和任务级模型 profile。
- AI 预算日用量和最近 AI 调用诊断的脱敏摘要。

桌面版 API Key 保存在系统 Keychain。SQLite、目录式备份、错误文案和 AI 调用诊断不应保存 API Key。

## What Is Sent To AI Providers

读伴只在你主动触发 AI 功能时发送请求，例如测试连接、生成整本书导读、生成章节导读、阅读中问答、选中文字提问、本书读伴聊天或读后交流。

根据具体功能，请求中可能包含：

- 你的 API Key，请求头会发送给当前模型服务商。
- 书名、作者、章节范围和阅读计划信息。
- 当前章节文本、当前页文本、全书抽样文本或你选中的原文。
- 本书读伴聊天所需的阅读进度、读伴记忆、整本书地图、当前阅读项少量摘录和最近沉淀摘要。
- 最近几条伴读问答、读后交流历史。
- 当你选择带入上下文时，相关高亮、笔记和 AI 回答摘录。

读伴不会把这些内容发送到读伴自己的服务器。模型服务商如何处理请求数据，以你选择的服务商政策为准。

## API Key And BYOK

读伴采用 BYOK（Bring Your Own Key）模式。API Key 由用户自己提供。

需要注意：

- 浏览器版 API Key 保存在当前浏览器 IndexedDB 中；IndexedDB 不是硬件级安全存储。
- 桌面版 API Key 保存在系统 Keychain 中；这比普通本地文件更适合保存密钥，但仍依赖可信的本机账户和系统环境。
- 桌面版设置页会显示本机是否已有已保存 Key，但不会在打开设置页时自动把 Keychain 密钥读回输入框。
- 不可信设备、浏览器扩展、同源页面脚本、本机恶意软件或能访问浏览器配置文件的人，可能读取本地数据。
- 下载当前 AI 配置会生成包含 API Key 的 TXT 文件，请只保存在可信位置。
- 建议为读伴使用单独 API Key，并在模型服务商后台设置额度、限额或调用范围。

## Custom Base URL

OpenAI-compatible 配置允许填写自定义 Base URL。测试连接和生成内容时，API Key 与必要阅读文本会发送到该地址。

读伴会对非官方或非 HTTPS 的 Base URL 做二次确认，但无法替你验证第三方服务是否可信。请只在你完全信任该服务商或本地代理时使用自定义地址。

## What Is Not Collected

当前版本不会主动收集：

- 账号、邮箱或身份信息。
- 设备指纹、广告标识或跨站追踪数据。
- 使用统计、遥测或产品分析事件。
- 书籍文件、笔记或聊天记录的云端副本。

桌面版最近 AI 调用诊断、本地诊断日志和诊断包仅保存在本机，用于帮助用户理解请求状态和排查问题。诊断只记录任务类型、供应商、模型、Base URL origin、耗时、状态、错误码、HTTP 状态、尝试次数、token、费用估算、App 启动、本地存储初始化、数据库健康检查和备份摘要；不记录 API Key、完整 prompt、章节正文、笔记正文或聊天全文。

## Data Control

你可以在设置页清空全部本地数据。清空后，书籍、进度、导读缓存、聊天记录、笔记和 API Key 会从对应本地存储中删除，且无法恢复。

浏览器版支持 JSON 备份导出/导入。桌面版支持目录式 `manifest.json + files/` 备份、导入前预览、校验报告、合并导入和覆盖恢复。备份默认不包含 API Key，也不会导出本机 Keychain 状态。

请不要把浏览器 IndexedDB 或单台电脑视为唯一备份。重要书库建议定期导出备份，并保存在可信位置。

## Third-Party Services

当你使用 Anthropic、OpenAI、DeepSeek、Kimi 或其他 OpenAI-compatible 服务时，请同时阅读对应服务商的隐私政策、数据使用政策和 API 条款。

读伴无法控制第三方模型服务商如何保存、处理或审查请求数据。
